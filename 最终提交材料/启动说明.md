# 立体多层停车场管理系统启动说明

## 一、项目环境

本项目使用 **C 语言** 编写，建议使用支持 C 语言编译的环境运行，例如：

- Windows + GCC / MinGW / MSYS2
- Linux + GCC
- macOS + Clang 或 GCC

项目代码分为三个目录：

```text
pzx/    数据结构基础层
 yjk/   停车业务与计费层
 zxc/   菜单界面、统计、文件读写和主程序
```

程序入口文件为：

```text
zxc/main.c
```

## 三、编译方法

请在项目根目录下执行编译命令。

项目根目录即包含 `pzx`、`yjk`、`zxc` 三个文件夹的目录。

### Windows / PowerShell / CMD

如果已安装 GCC，可执行：

```bash
gcc -I pzx -I yjk -I zxc pzx/ds.c yjk/parking.c yjk/fee.c zxc/stats.c zxc/ui.c zxc/file_io.c zxc/main.c -o parking.exe
```

如果想开启编译警告检查，可执行：

```bash
gcc -Wall -Wextra -I pzx -I yjk -I zxc pzx/ds.c yjk/parking.c yjk/fee.c zxc/stats.c zxc/ui.c zxc/file_io.c zxc/main.c -o parking.exe
```

### Linux / macOS

```bash
gcc -Wall -Wextra -I pzx -I yjk -I zxc pzx/ds.c yjk/parking.c yjk/fee.c zxc/stats.c zxc/ui.c zxc/file_io.c zxc/main.c -o parking
```

---

## 四、运行方法

### Windows

编译成功后，运行：

```bash
.\parking.exe
```

### Linux / macOS

编译成功后，运行：

```bash
./parking
```

## 五、程序菜单说明

程序启动后会显示主菜单：

```text
1. 车辆入场登记
2. 车辆出场结算
3. 查找车辆
4. 空闲车位查看
5. 等待队列查看
6. 车位状态总览
7. 收入统计
8. 历史交易明细
9. 楼层热度分析
10. 峰值时段分析
11. 保存数据
12. 加载数据
0. 退出系统
```

根据提示输入对应数字即可使用功能。

例如：

- 输入 `1`：登记车辆入场
- 输入 `2`：办理车辆出场并结算费用
- 输入 `3`：根据车牌号查询车辆
- 输入 `0`：退出系统

## 六、数据保存说明

程序运行过程中会使用文件：

```text
parking_data.dat
```

该文件用于保存停车场运行数据。

- 第一次运行时，如果没有该文件，程序会自动使用初始状态。
- 退出程序时，系统会自动保存数据。
- 菜单中也可以选择“保存数据”或“加载数据”。

## 七、常见问题

### 1. 提示找不到 `ds.h`

请确认编译命令中包含：

```bash
-I pzx
```

完整编译命令示例：

```bash
gcc -I pzx -I yjk -I zxc pzx/ds.c yjk/parking.c yjk/fee.c zxc/stats.c zxc/ui.c zxc/file_io.c zxc/main.c -o parking.exe
```

### 2. 提示找不到 `parking.h` 或 `fee.h`

请确认编译命令中包含：

```bash
-I yjk
```

### 3. 提示找不到 `ui.h`、`stats.h` 或 `file_io.h`

请确认编译命令中包含：

```bash
-I zxc
```

### 4. Windows 下提示 `gcc` 不是内部或外部命令

说明当前电脑没有安装 GCC，或者 GCC 没有加入环境变量。

可以安装以下任一工具：

- MinGW-w64
- MSYS2
- Dev-C++ 自带 GCC
- Code::Blocks 自带 GCC

安装后重新打开终端，再执行编译命令。

```bash
# 1. 进入项目根目录

# 2. 编译程序

# 3. 启动程序
.\parking.exe
```

如果是在 Linux 或 macOS 下，将最后一行改为：

```bash
./parking
```
